\hypertarget{README.md_autotoc_md172}{}\doxysubsection{\texorpdfstring{Introduction}{Introduction}}\label{README.md_autotoc_md172}
\href{https://github.com/i18next/i18next-http-backend/actions?query=workflow\%3Anode}{\texttt{}} \href{https://github.com/i18next/i18next-http-backend/actions?query=workflow\%3Adeno}{\texttt{}} \href{https://www.npmjs.com/package/i18next-http-backend}{\texttt{}}

This is a simple i18next backend to be used in Node.\+js, in the browser and for Deno. It will load resources from a backend server using the XMLHttp\+Request or the fetch API.

Get a first idea on how it is used in \href{https://youtu.be/SA_9i4TtxLQ?t=953}{\texttt{this i18next crash course video}}.

It\textquotesingle{}s based on the deprecated \href{https://github.com/i18next/i18next-xhr-backend}{\texttt{i18next-\/xhr-\/backend}} and can mostly be used as a drop-\/in replacement.

{\itshape \href{https://github.com/i18next/i18next-xhr-backend/issues/348\#issuecomment-663060275}{\texttt{Why i18next-\/xhr-\/backend was deprecated?}}}\hypertarget{README.md_autotoc_md173}{}\doxysubsection{\texorpdfstring{Advice\+:}{Advice\+:}}\label{README.md_autotoc_md173}

{\bfseries{Seeing failed http requests, like 404?}}

Are you using a \href{https://github.com/i18next/i18next-browser-languageDetector}{\texttt{language detector}} plugin that detects region specific languages you are not providing? i.\+e. you provide `\textquotesingle{}en'{\ttfamily  translations but you see a }\textquotesingle{}en-\/\+US\textquotesingle{}\`{} request first?

This is because of the default {\ttfamily load} \href{https://www.i18next.com/overview/configuration-options}{\texttt{option}} set to `\textquotesingle{}all'\`{}.

Try to set the {\ttfamily load} \href{https://www.i18next.com/overview/configuration-options}{\texttt{option}} to `\textquotesingle{}language\+Only'\`{}
